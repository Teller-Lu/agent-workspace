# Sample（示例项目）

> 本项目是 **agent-workspace 的示例项目**，演示典型项目的文件构造与待办用法。
> 正式使用时：复制本目录改名为你的项目，把示例待办替换为真实内容，并在根级 `AGENTS.md` §七 的"项目缩写登记表"登记缩写。
> 项目入口文件（本文件）为 **L1**，修改需用户批准 + 审计。

---

## 一、项目信息

| 项 | 值 |
|---|---|
| 项目名 | Sample |
| 缩写 | `SP` |
| 入口 | `AGENTS.md`（本文件） |
| 状态 | 示例（供参考） |

---

## 二、项目规则

1. 启动顺序：先读根级 `../AGENTS.md`，再读本文件。
2. 待办统一放 `待办.md`（标准模板，见根级 `AGENTS.md` §七）；编号用 `SP-NNN`。
3. 需求/设计/计划文档开发中为 L2，标记"已完成"后升 L1。
4. 子项目内不擅自 `git push`，交由根级 git-agent。

---

## 三、典型文件构造（供后来者参考）

```text
Sample/
├── AGENTS.md          # 项目入口（L1）：项目信息、规则、文件构造说明
├── README.md          # 给人看的项目说明
├── 待办.md            # 标准待办模板（待办总表 + 已完成），编号 SP-NNN
├── 需求.md            # （按需）需求清单与验收标准，开发中 L2 / 已完成 L1
├── 设计.md            # （按需）设计方案
└── src/ 或 Scripts/   # （按需）代码/脚本，开发中 L2 / 已完成 L1
```

不是每个文件都必须有——按项目需要建。`AGENTS.md` + `待办.md` 是最小集。
